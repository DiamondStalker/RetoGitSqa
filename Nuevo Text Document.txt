# Reto POM - Control de Versiones

¡Bienvenido al reto POM! Este desafío tiene como objetivo poner en práctica los conocimientos adquiridos sobre control de versiones y gestión de repositorios, en especial con Git y GitHub. En este proyecto, demostrarás tu habilidad para aplicar los conceptos y técnicas vistas en el curso para manejar archivos, ramas, commits y etiquetas en un flujo de trabajo real.

## Objetivo

El objetivo de este reto es validar y demostrar tus conocimientos sobre control de versiones al crear, organizar y gestionar archivos y commits dentro de un repositorio, incluyendo la configuración de ramas y etiquetas. 

## Descripción del Reto

Este reto consta de una serie de pasos secuenciales que debes seguir para cumplir los criterios de calificación:

### 1. Crear un Proyecto Local
- Crea un proyecto cualquiera en tu equipo local.
- Prepara los archivos para el primer commit y realiza el commit inicial con el mensaje:

```bash
commit 1 para el reto
```


### 2. Añadir un Archivo Nuevo
- Crea una nueva clase o archivo en el proyecto.
- Añádelo al repositorio local y realiza un segundo commit con el mensaje:

```bash
commit 2 para el reto
```

### 3. Crear y Trabajar en una Rama
- Crea una nueva rama llamada `rama2`.
- Ubícate en esta rama, crea un nuevo archivo o clase, guarda los cambios y realiza un commit en esta rama con el mensaje:

```bash
commit 3 rama2 para el reto
```

### 4. Merge y Creación de Tag
- Vuelve a la rama principal (`main`) y realiza un merge con `rama2`.
- Crea un tag con la versión `v1.0` del repositorio y agrega un mensaje personalizado que describa esta versión.

### 5. Subir el Repositorio a GitHub
- Crea un repositorio público en GitHub y añade este repositorio como remoto en tu repositorio local.
- Sube las dos ramas (`main` y `rama2`) y el tag (`v1.0`) al repositorio remoto en GitHub.
